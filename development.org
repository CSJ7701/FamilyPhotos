* Project Overview
Create a containerized, Python-based web application to serve as a central "landing zone" for family updates and Immich-integrated photo viewing.

* Technical Requirements
** Backend
- *Language:* Python 3.11+
- *Framework:* FastAPI (recommended for high-performance async API calls to Immich) or Flask.
- *Auth:* OAuth2 or session-based authentication with a local SQLite/PostgreSQL database for user management.
- *Integration:* Immich REST API (using ~python-requests~ or ~httpx~).

** Frontend
- *Templating:* Jinja2 (server-side rendering) or a lightweight JS framework.
- *Styling:* CSS Grid/Flexbox or Tailwind CSS for a responsive layout.
- *Carousel:* Swiper.js or a simple CSS-based scroll snapper.

** Infrastructure
- *Containerization:* Docker & Docker Compose.
- *Persistence:* Docker volumes for the user database and local configuration files.

* Functional Requirements
** Public Landing Page
- [ ] *Announcements Section:* Markdown-based text or simple DB-stored strings.
- [ ] *Immich Carousel:* Fetch images from a specific ~ALBUM_ID~ via the Immich API.
- [ ] *Flavor Text:* Configurable header/footer text for a "homey" feel.
- [ ] *Navigation Bar:* Context-aware links (Login/Logout, Photos, Profile).

** User Management
- [ ] *Account Requests:* Unauthenticated users can submit a request (Username, Email, Reason).
- [ ] *Authentication:* Login/Password hashing (Argon2 or bcrypt).
- [ ] *Admin Dashboard:*
  - Approve/Deny pending account requests.
  - Configure Immich API URL and default Album ID for carousel.
  - Configure default album for photo upload.
  - Manage existing users.

** Photos Page (Protected)
- [ ] *Authorization:* Redirect to login if no active session.
- [ ] *Album List:* Pull all public/shared albums from the Immich instance.
- [ ] *Photo Upload*: 'Upload' button should allow users to upload local photos to the Immich server. These should be placed into an admin specified album within Immich for sorting later.

* Development Process
** Phase 1: Environment & Immich Integration
1. Initialize Git repository.
2. Draft ~docker-compose.yml~ with Python app and PostgreSQL/SQLite.
3. Create a basic FastAPI/Flask app that successfully authenticates with the Immich API using an API Key.
4. Implement the "Featured Carousel" logic on a boilerplate homepage.

** Phase 2: Authentication System
1. Define SQLAlchemy/SQLModel schemas for ~Users~ and ~AccountRequests~.
2. Build the "Request Account" form.
3. Implement session-based login and a basic Admin role check.
4. Build the Admin view to toggle user "Active" status.

** Phase 3: UI/UX Refinement
1. Build the Navbar with conditional rendering based on session state.
2. Implement the Photos page using a grid layout.
3. Style the landing page for a clean, "hub" aesthetic.

** Phase 4: Hardening & Deployment
1. Environment variable management (Immich URL, API Key, Secret Keys).
2. Optimize Dockerfile (multi-stage builds for smaller images).
3. (Optional) Set up a reverse proxy (Caddy/Nginx) for HTTPS.
